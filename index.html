💡 Grundprinzip: Erst verstehen, dann anwenden. Nicht gleichzeitig. Volle Führung am Anfang ist die Voraussetzung für echte Selbstständigkeit am Ende. Wer Lernende zu früh alleine lässt, produziert Beschäftigung statt Lernen.

Das System unterscheidet zwei Einheitentypen, die sich gegenseitig ergänzen:

Lerneinheit

Thema verstehen

Neuen Stoff erschließen, Konzepte aufbauen

  • Vollständiger Guided Learning Cycle
  • Differenzierung BASIS / STANDARD / CHALLENGE
  • Scaffolding und Hilfekarten
  • Fehler-Radar zur gemeinsamen Reflexion
Kompetenztraining

Prüfungsfit werden

Vorbereitung auf Leistungsnachweis und Abschlussprüfung

  • Prüfungsbedingungen konsequent simulieren
  • Echte Aufgaben aus vergangenen Prüfungen
  • Kein Scaffolding, alle SuS gleiche Aufgabe
  • Ausführlicher Fehler-Radar im Anschluss
Die 3 Werkzeuge

Drei Tools. Ein System.

Diese drei Tools machen strukturierten Unterricht umsetzbar, nicht nur theoretisch schön. Sie sind die praktische Grundlage für den Guided Learning Cycle.

Werkzeug für Lernende

Lernkompass

Wiederverwendbare Struktur statt Einweg-Arbeitsblätter. Jede Phase hat ihr eigenes Feld.

  • Lernende wissen immer, wo sie gerade stehen
  • Sicherungsfenster in Phase A direkt eingebaut
  • Fehler-Radar und Exit Ticket als feste Felder
  • Auch als PWA-App verfügbar (iOS/Android)

Präsentations-Tool

Gamma

Schnelle Materialerstellung: professionelle Folien in Minuten statt Stunden.

  • Visualisierungen, Aufgaben, klare Strukturen
  • Kein Formatierungswahnsinn mehr
  • FOSBOS-Theme automatisch angewendet
  • Label-Badges: AKTIVIERUNG, INPUT-CHUNK, MINI-CHECK ...
  • Direkt vom KI-Workflow hochgeladen, kein Copy-Paste
📎 Begleitmaterial direkt im Deck
KI-Workflow

KI-Unterstützung

Differenzierung praktikabel machen: automatisch, vollständig, fachlich korrekt.

  • Generiert automatisch 3 Niveaus (BASIS / STANDARD / CHALLENGE)
  • Liest Lehrplan und Abituraufgaben automatisch
  • Fachcheck vor dem GO: kein halluzinierter Output
  • 8 spezialisierte Komponenten (Skills, Agents, APIs)
  • Kein Prompt in ChatGPT, ein echtes Automatisierungssystem

Das steckt wirklich dahinter

Der KI-Workflow im Detail

8 spezialisierte Komponenten arbeiten zusammen. Kein generischer Chatbot, sondern ein System das den Lehrplan kennt, Abituraufgaben analysiert und fachlich prüft.

Nicht ChatGPT. Ein System.

Ein einzelner Prompt in ChatGPT liefert generischen Output ohne Lehrplanbezug, ohne Differenzierung, ohne fachliche Prüfung. Dieser Workflow koordiniert 8 spezialisierte Komponenten, jede mit einer eigenen Aufgabe, alle auf das bayerische Schulsystem ausgerichtet.

Super-Skill
① Didaktik-Skill — das Gehirn des Systems
Lehrplan lesen · Abitur-Frequenz-Check · Hook-Generator · Konzeptvorschlag

Das zentrale Steuerelement des gesamten Workflows. Als einzige Komponente koordiniert er alle anderen vollständig: vom Lehrplan-Lesen über den Fachcheck bis zum fertigen Gamma-Upload. Liest Lehrplan-PDFs aller Fächer und scannt aktuelle Abituraufgaben. Analysiert: Kommt dieses Thema im FAP vor? Wie häufig? Daraus folgt: Rechenschema-Drilling JA oder NEIN. Generiert 3 Hook-Optionen, regional verankerten Leitfall, vollständige Differenzierung. Wartet auf GO der Lehrkraft.

→ Recherche-Agent → Fachcheck-Agent → Grafiken-Skill (opt.) → Kartenset-Agent
Agent
② Recherche-Agent
bei Thema ohne URL · auch zur laufenden Literaturrecherche

Sucht 3 bis 5 passende Quellen parallel: aktuelle Artikel, Fachliteratur, statistische Daten. Bewertet didaktische Eignung, extrahiert Schlüsselzahlen und Aktualität. Empfiehlt die beste Quelle und verwirft ungeeignete. Kann auch unabhängig von einer konkreten Anfrage passende Materialien zu einem Thema zusammenstellen.

Agent
③ Fachcheck-Agent
immer aktiv · vor dem GO

Prüft das Konzept auf fachliche Vollständigkeit: fehlende Inhalte, falsche Begriffe, LN-Relevanz. Bewertet jeden Punkt als HOCH / MITTEL / NIEDRIG. Das Konzept wird angepasst, bevor GO abgefragt wird.

Optional · Skill
④ Unterrichts-Grafiken-Skill
bei Grafikbedarf im Konzept · für alle Fächer

Erstellt professionelle SVG-Grafiken: Angebot-Nachfrage-Diagramme, Elastizitätskurven, Metaphernbilder, Strukturdiagramme. Konvertiert automatisch zu PNG für den direkten Gamma-Upload. Wird vom Didaktik-Skill aufgerufen, wenn das Konzept Grafikbedarf erkennt.

⋯ nur aufgerufen wenn Konzept Grafikbedarf erkannt hat
▶ GO — Lehrkraft bestätigt das Konzept
Agent
⑤ Gamma-Kartenset-Agent
nach GO · vollständiger Guided Learning Cycle

Generiert ein vollständiges Kartenset in der richtigen Phasenreihenfolge: Einstieg, Strukturierter Input (bis 4 Chunks mit Mini-Checks), Geführte Anwendung (Leitfall mit Differenzierung und Lösungskarten), Konsolidierung mit Fehler-Radar. Alle Strukturhinweise als Label-Badges, Infomaterial-Karten als Begleitmaterial direkt im Deck eingebettet.

API
⑥ Gamma API — vollautomatischer Upload
FOSBOS-Theme · Label-Badges automatisch · sofortiger Link · kein Copy-Paste

Das Kartenset wird direkt via Gamma API hochgeladen. Das FOSBOS-Theme wird automatisch angewendet, Label-Badges werden auf jeder Karte korrekt platziert. Keine manuelle Nachbearbeitung. Ergebnis: fertiger Gamma-Link in Sekunden.

Skill
⑦ Lehrerhandbuch-Skill
nach Gamma-Upload · HTML · im Browser abrufbar

Erstellt eine HTML-Datei für die Lehrkraft, direkt im Browser abrufbar, ohne Login und ohne App-Download. Vollständiger Stundenverlaufsplan mit Kartennummern, Musterlösungen (standardmäßig verborgen, per Klick öffenbar), Fehler-Radar mit WARUM-Block, Satzstarter, Impulsfragen, Differenzierungshinweise.

→ Direkt geschrieben→ Browser-Aufruf

Gedächtnis
⑧ Persistentes Gedächtnis-System
automatisch aktualisiert · kein Kontextverlust zwischen Sessions

Nach jeder Session wird der Projektstand automatisch gesichert: Gamma-Links, Leitfall-Zahlen, Sequenz-Arc-Fortschritt, offene Baustellen. Beim nächsten Chat wird der Stand automatisch geladen. Das System weiß, wo es aufgehört hat. Kein Neustart von null.
